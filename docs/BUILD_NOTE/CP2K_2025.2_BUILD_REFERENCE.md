## 3. 配置文件详解

### 3.1 `configs/versions.yaml`

```yaml
# HPC Container Build Configuration

images:
  builder_base: debian:trixie
  runtime_base: debian:trixie-slim

spack:
  version: "1.1.0"
  use_mirror: false
  mirror_path: "/opt/spack-mirror"

applications:
  cp2k:
    version: "2025.2"
    enabled: true
    spack_env: "spack-envs/cp2k-opensource-2025.2"

features:
  gpu_support: false

build:
  jobs: 4
  verbose: true
  cache: true
```

**关键参数说明：**

| 参数 | 值 | 说明 |
|---|---|---|
| `images.builder_base` | `debian:trixie` | 构建阶段基础镜像 |
| `images.runtime_base` | `debian:trixie-slim` | 运行阶段基础镜像（精简） |
| `spack.version` | `1.1.0` | Spack 版本，对应 `assets/spack-v1.1.0.tar.gz` |
| `applications.cp2k.version` | `2025.2` | CP2K 版本号 |
| `applications.cp2k.spack_env` | `spack-envs/cp2k-opensource-2025.2` | Spack 环境文件路径 |

### 3.2 `spack-envs/cp2k-opensource-2025.2/spack.yaml`

#### 全局包偏好设置

```yaml
packages:
  all:
    buildable: true
    prefer:
      - ~cuda          # 禁用 CUDA
      - +mpi           # 启用 MPI
      - +openmp        # 启用 OpenMP
      - +pic           # 位置无关代码
      - ~rocm          # 禁用 ROCm
      - target="x86_64_v3"  # 目标架构
```

#### MPI 配置

```yaml
mpi:
  require:
    - openmpi
openmpi:
  require:
    - +internal-hwloc
```

#### 线性代数库配置

```yaml
blas:
  require:
    - openblas
lapack:
  require:
    - openblas
openblas:
  require:
    - +dynamic_dispatch    # 动态调度（运行时检测 CPU）
    - +fortran             # Fortran 接口
    - threads=openmp       # OpenMP 线程
scalapack:
  require:
    - netlib-scalapack
```

## 4. Dockerfile 生成

### 4.1 生成命令

```bash
cd /home/shaojiehe/HPC-Container-Factory

# 生成 Dockerfile（使用 --mirror 启用离线源码镜像，--dry-run 仅生成不构建）
python generate.py \
    --config configs/versions.yaml \
    --template templates/Dockerfile-cp2k-opensource-2025.2.j2 \
    --output Dockerfile \
    --mirror \
    --dry-run
```

### 4.2 `generate.py` 完整参数说明

| 参数 | 本次使用的值 | 默认值 | 说明 |
|---|---|---|---|
| `--config` | `configs/versions.yaml` | `configs/versions.yaml` | YAML 配置文件路径 |
| `--template` | `templates/Dockerfile-cp2k-opensource-2025.2.j2` | 自动检测 | Jinja2 模板路径 |
| `--output` | `Dockerfile` | `Dockerfile` | 输出文件路径 |
| `--mirror` | ✅ 启用 | 未启用 | 使用本地 Spack 离线镜像 |
| `--dry-run` | ✅ 启用 | 未启用 | 仅生成 Dockerfile，不执行构建 |
| `--build` | 未使用 | 未启用 | 生成后立即构建 |
| `--build-only` | 未使用 | 未启用 | 仅生成 builder 阶段 |
| `--app` | `cp2k` | `cp2k` | 应用类型 |
| `--app-version` | `opensource-2025.2` | `opensource-2025.2` | 应用版本 |
| `--builder` | 未使用 | `docker` | 构建后端（docker/apptainer） |
| `--image` | 未使用 | `hpc-cp2k` | 镜像名称 |
| `--tag` | 未使用 | `latest` | 镜像标签 |
| `--verbose` | 未使用 | 未启用 | 详细日志 |

### 4.3 Jinja2 模板变量

`generate.py` 从 `versions.yaml` 和命令行参数构建模板上下文：

| 模板变量 | 来源 | 值 |
|---|---|---|
| `{{ builder_base_image }}` | `versions.yaml → images.builder_base` | `debian:trixie` |
| `{{ runtime_base_image }}` | `versions.yaml → images.runtime_base` | `debian:trixie-slim` |
| `{{ use_mirror }}` | CLI `--mirror` | `true` |
| `{{ build_only }}` | CLI `--build-only` | `false` |
| `{{ timestamp }}` | `datetime.now().isoformat()` | `2026-04-12T05:20:47.150347` |

## 5. 构建执行

### 5.1 Podman 构建命令

```bash
cd /home/shaojiehe/HPC-Container-Factory

podman build \
    --network=host \
    -f Dockerfile \
    -t hpc-cp2k:2025.2 \
    .
```

**参数说明：**

| 参数 | 说明 |
|---|---|
| `--network=host` | 使用宿主网络（拉取 git 仓库和 apt 包） |
| `-f Dockerfile` | 指定生成的 Dockerfile |
| `-t hpc-cp2k:2025.2` | 镜像名称和标签 |
| `.` | 构建上下文为项目根目录 |

### 5.2 构建阶段详解

整个构建分为 **3 个阶段**（multi-stage build）：

#### Stage 1: `builder`（编译环境）

| 步骤 | 描述 | 关键参数 |
|---|---|---|
| APT 镜像 | 切换到中科大 USTC 镜像 | `mirrors.ustc.edu.cn` |
| 系统依赖 | 安装编译工具链 | `build-essential, gfortran, cmake, ninja-build, python3-dev` 等 30+ 包 |
| Spack 安装 | 从本地 tar.gz 解压 | `assets/spack-v1.1.0.tar.gz` → `/opt/spack-exe/` |
| Bootstrap 配置 | 注册本地 bootstrap 源 | `spack bootstrap add --trust local-sources/binaries` |
| 离线镜像 | 绑定挂载源码镜像 | `--mount=type=bind,source=assets/spack-mirror,target=/opt/spack-mirror,readonly` |
| CP2K 源码 | Git clone | `git clone --depth 1 --recursive -b support/v2025.2 https://github.com/cp2k/cp2k.git /opt/cp2k` |
| Spack 环境 | 创建环境 | `spack env create cp2k-env` |
| 编译器发现 | 自动检测 GCC | `spack compiler find` |
| 外部包发现 | 检测系统包 | `spack external find --all --not-buildable` |
| CP2K 仓库 | 添加官方 repo | `spack repo add .../cp2k_dev_repo/` |
| Concretize | 锁定依赖版本 | `spack -e cp2k-env concretize -f` |
| zlib 修复 | 符号链接 | `ln -sf /usr/lib/x86_64-linux-gnu/libz.so /usr/lib/libz.so` |
| cmake 脚本修复 | 修改构建参数 | `sed -i -e 's/myenv/cp2k-env/g' -e '/-DCP2K_USE_LIBXSMM=OFF/d' -e '/-DCP2K_USE_TBLITE=OFF/d'` |
| 依赖编译 | depfile + make | `spack -e cp2k-env env depfile -o ... && make -j $(nproc)` |
| 视图提取 | 复制到 /opt/spack | `cp -ar ${SPACK_ENV_VIEW}/{bin,include,lib} /opt/spack` |
| CMake 配置 | CP2K cmake | `source ./cmake/cmake_cp2k.sh spack_all psmp` |
| CP2K 编译 | ninja 并行编译 | `ninja --verbose` + `ninja --verbose install` |
| Strip | 去除调试符号 | `find /opt/{spack,cp2k} -type f -exec strip {} \;` |

#### Stage 2: `runtime`（运行时）

| 步骤 | 描述 |
|---|---|
| 基础镜像 | `debian:trixie-slim` |
| 运行时依赖 | `ca-certificates, libc6, openssh-client, tar, perl, nano, curl, python3, bash, g++, gcc, gfortran, screen, util-linux` |
| 用户创建 | `useradd -m -s /bin/bash hpc` |
| 文件复制 | 从 builder 复制 `/opt/spack`, `/opt/cp2k/{bin,lib,share,tests}` |
| ldconfig | 注册库搜索路径 `/opt/cp2k/lib`, `/opt/spack/lib` |
| 清理 | 删除所有 `.a` 静态库 |
| 工作目录 | `/workspace` |
| 运行用户 | `hpc`（非 root） |

#### Stage 3: `final`（别名）

- 纯别名，指向 `runtime`，用于向后兼容

### 5.3 运行时环境变量

```bash
# 容器内环境变量
PATH="/opt/spack/bin:/opt/cp2k/bin:/opt/bin:$PATH"
LD_LIBRARY_PATH="/opt/spack/lib:/opt/lib:$LD_LIBRARY_PATH"
CP2K_DATA_DIR="/opt/share/cp2k/data"
SPACK_ROOT="/opt/spack-exe"
```

## 9. 故障排除

### 9.1 "executable file not found" 错误

**现象：** `podman run --rm localhost/hpc-cp2k:2025.2 cp2k.psmp --version` 返回
`Error: crun: executable file ... not found`

**原因：** 未指定 `--user hpc` 时，容器以 root 用户运行，动态链接器路径可能不正确。

**解决：** 始终使用 `--user hpc` 运行：

```bash
podman run --rm --user hpc localhost/hpc-cp2k:2025.2 cp2k.psmp --version
```

### 9.2 构建时间过长

- 完整构建（含所有依赖）约需 **45-90 分钟**（取决于机器性能和网络）
- 依赖编译（`make -j $(nproc)`）占大部分时间
- `libint@2.9.0` 和 `py-torch@2.7` 是编译时间最长的两个包

### 9.3 磁盘空间需求

- 构建中间层可达 **6+ GB**
- 最终镜像 **3.02 GB**
- 建议至少预留 **20 GB** 磁盘空间用于完整构建
