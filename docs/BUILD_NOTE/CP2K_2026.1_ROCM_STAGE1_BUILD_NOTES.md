## 0. 概述

### 目标

为 **AMD MI300X (gfx942)** 构建 **CP2K 2026.1 GPU 加速容器镜像**的 Spack 依赖层。

### 构建策略

| 阶段 | 工具 | 构建内容 | 说明 |
|------|------|----------|------|
| **Stage 1** | Spack 1.1.0 | CPU 依赖 + DLAF GPU 链 (~60 包) | Spack 版本管理精确，自动处理依赖树 |
| **Stage 2** | CP2K toolchain | HIP 库 (DBCSR) + CP2K cmake | toolchain 已有成熟 HIP patch |

### 关键技术决策

1. **DLAF 替代 ELPA GPU** — DLA-Future 基于 pika 异步运行时，无 GPU+OMP 冲突；ELPA `+rocm+openmp` 有 SIGSEGV (cp2k#464)
2. **ELPA 已删除** — 不再安装 ELPA，DLAF 作为唯一 GPU 特征值求解器后端
3. **混合编译器** — CPU 包用 `%gcc@13.3.0`，DLAF 链（C++/HIP）用 `%llvm-amdgpu@7.2.1`（amdclang）
4. **不含 SIRIUS** → 跳过平面波支持 → 消除 SPLA/SpFFT HIP 构建需求
5. **MPI** → mpich@4.3.2，与官方 ROCm CI 一致

## 1. Container Setup

### Base Image
```
docker.io/rocm/dev-ubuntu-24.04:7.2.1-complete
```
- **Must be `-complete` variant**: includes hip, rocblas, rocsolver, hipblas, llvm-amdgpu etc.
- Ubuntu 24.04 (noble), uses **new-style** `/etc/apt/sources.list.d/ubuntu.sources`

### Container Creation
```bash
podman run -d --name cp2k-stage1 \
  --network=host --userns=keep-id \
  -v "$(pwd)":/work:Z \
  docker.io/rocm/dev-ubuntu-24.04:7.2.1-complete \
  sleep infinity
```

> ⚠️ Container needs `-u root` for all exec commands (otherwise runs as non-root due to `--userns=keep-id`)

## 3. Spack Installation + Mirror Configuration

```bash
podman exec -u root cp2k-stage1 bash -c '
cd /opt
git clone --depth 1 --branch v1.1.0 https://github.com/spack/spack.git spack-exe
source /opt/spack-exe/share/spack/setup-env.sh

# Bootstrap mirror (offline)
spack bootstrap add --trust local-sources /work/assets/bootstrap/metadata/sources

spack bootstrap disable github-actions-v0.6
spack bootstrap disable spack-install

# Source mirror
spack mirror add --scope site local-mirror file:///work/assets/spack-mirror
'
```

- Spack 1.1.0 at `/opt/spack-exe`
- Bootstrap provides: gcc-runtime, patchelf, clingo-bootstrap
- Source mirror at `/work/assets/spack-mirror`

## 5. Environment Setup

```bash
podman exec -u root cp2k-stage1 bash -c '
source /opt/spack-exe/share/spack/setup-env.sh
mkdir -p /opt/hpc
cp /work/spack-envs/cp2k-2026.1-rocm/spack.yaml /opt/hpc/

# ⚠️ blt compatibility symlink (ROCm 7.x removed /opt/rocm/hip)
ln -sf /opt/rocm /opt/rocm/hip
'
```

> ⚠️ **不要**复制 `spack.lock`！每次 concretize 需要根据当前容器环境重新解析。

### 关键 spack.yaml 配置要点

```yaml
packages:
  all:
    prefer:
      - ~rocm              # 默认不用 ROCm
      - target="x86_64_v3"
  libint:
    require:
      - "%gcc@13.3.0"      # amdflang 有 MPI Fortran module 问题
  dftd4/tblite/mctc-lib/...:
    require:
      - "%gcc@13.3.0"      # amdflang 不兼容 -Mbackslash flag
  dla-future-fortran:
    require:
      - "+rocm"
      - "dla-future+rocm+scalapack amdgpu_target=gfx942"
  umpire:
    require:
      - "@2024.07.0:"      # 旧版缺少 PoolCoalesceHeuristic API
  hip/rocblas/rocsolver/llvm-amdgpu/hsa-rocr-dev:
    buildable: false       # 使用 ROCm 7.2.1 base image 自带
```

## 7. Issues & Solutions

| Issue | Solution | Notes |
|-------|----------|-------|
| APT 502 Bad Gateway | Switch to USTC mirror | Ubuntu 24.04 uses `.sources` format |
| Non-root in container | Use `podman exec -u root` | Caused by `--userns=keep-id` |
| Bootstrap needs offline | local bootstrap + disable online | Critical for air-gapped builds |
| `SPACK_CXX=None` | `spack compiler find` auto-detects `extra_attributes.compilers` | Don't manually write ROCm compiler config |
| amdflang `use mpi` fail | **Force `%gcc@13.3.0` for Fortran packages** (libint, dftd4 chain) | amdflang can't compile `use mpi` Fortran module with mpich |
| amdflang `-Mbackslash` | Force `%gcc@13.3.0` for mctc-lib chain | amdflang identified as Flang, triggers PGI-style flags |
| blt `FindHIP.cmake` ROCm 7.x | `ln -s /opt/rocm /opt/rocm/hip` | ROCm 7.x removed `/opt/rocm/hip` dir |
| ~~ELPA `assert` undeclared~~ | ~~cassert patch~~ | **已废弃：ELPA 已删除** |
| ~~ELPA GPU+OMP SIGSEGV~~ | ~~DLAF 替代~~ | **已废弃：ELPA 已删除** |

## 9. Stage 1 → Stage 2 交接

| 产出 | 路径 | 说明 |
|------|------|------|
| Spack view | `/opt/hpc/view/` | 所有库的 copy（非 symlink） |
| mpich | `/opt/hpc/view/bin/mpicc` 等 | MPI 编译器 wrapper |
| DLAF | `/opt/hpc/view/lib/libdla_future*.so` | GPU 特征值求解器 |
| ROCm 库 | `/opt/rocm/lib/` | hip, rocblas, rocsolver 等 |
| spack.lock | `/opt/hpc/spack.lock` | 可导出回工作区 |

Stage 2 消费方式：
- `CP2K_USE_DLAF=ON` + `find_package(DLAFFortran 0.4.0)`
- `CMAKE_PREFIX_PATH` 包含 `/opt/hpc/view`
- toolchain 编译 DBCSR (HIP)
